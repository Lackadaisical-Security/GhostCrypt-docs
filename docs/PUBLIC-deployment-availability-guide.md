## Overview

This public deployment guide replaces operational deployment instructions with a non-operational
overview suitable for a public documentation repository. It does not include installation commands,
enterprise rollout scripts, cloud templates, license-generation commands, hardening scripts, endpoint
configuration details, or monitoring logic.

## Public Deployment Model

GhostCrypt deployment is handled only through authorized commercial channels. Deployment scope may
vary based on license tier, customer type, jurisdiction, end use, end user, and export-control review.

## Evaluation Process

A typical authorized evaluation may include:

1. initial licensing inquiry
2. end-user and end-use review
3. sanctions and restricted-party screening
4. export-control classification review where required
5. written agreement and scope definition
6. controlled delivery of documentation and/or binaries
7. deployment support through approved channels

## Administrative / Master Tooling

Administrative tooling, master tooling, license-generation infrastructure, and deployment automation
are private. They are not included in public repositories and are not available except through explicit
written authorization and compliance review.

## Public Support Boundary

Public deployment questions should be limited to availability, licensing, compliance, security contact,
and documentation clarification. Operational deployment assistance is limited to authorized customers
or approved evaluators.
